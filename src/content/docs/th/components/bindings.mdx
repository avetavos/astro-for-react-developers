React ใช้ **controlled input**: คุณตั้งค่า `value={state}` แล้วอัปเดตมันเองใน `onChange` นี่เป็นความตั้งใจ — one-way data flow ทำให้การเปลี่ยน state ชัดเจน ส่วน Svelte มี `bind:value` สำหรับ **two-way binding** ที่เชื่อม input กับตัวแปรเข้าด้วยกันโดยอัตโนมัติ ไม่มีอะไรใน React ที่เทียบได้ตรง ๆ — มันเป็น superpower ของ Svelte จริง ๆ

## Text input: controlled เทียบกับ `bind:value`

export const bindingsTextReact = `function NameForm() {
  const [name, setName] = React.useState('');
  return (
    <>
      <input
        value={name}
        onChange={e => setName(e.target.value)}
        placeholder="Your name"
      />
      <p>Hello, {name || 'stranger'}!</p>
    </>
  );
}`;

export const bindingsTextSvelte = `<script>
  let name = \$state('');
<\/script>

<input bind:value={name} placeholder="Your name" />
<p>Hello, {name || 'stranger'}!</p>`;

<ReactSvelte react={bindingsTextReact} svelte={bindingsTextSvelte} />

`bind:value` แทนที่ทั้ง `value={name}` และ `onChange={e => setName(e.target.value)}` ด้วย directive เดียว

## Checkbox: `bind:checked`

export const bindingsCheckReact = `function Opt() {
  const [agreed, setAgreed] = React.useState(false);
  return (
    <label>
      <input
        type="checkbox"
        checked={agreed}
        onChange={e => setAgreed(e.target.checked)}
      />
      I agree
    </label>
  );
}`;

export const bindingsCheckSvelte = `<script>
  let agreed = \$state(false);
<\/script>

<label>
  <input type="checkbox" bind:checked={agreed} />
  I agree
</label>
<p>Agreed: {agreed}</p>`;

<ReactSvelte react={bindingsCheckReact} svelte={bindingsCheckSvelte} />

## Select / dropdown

export const bindingsSelectReact = `function Picker() {
  const [color, setColor] = React.useState('red');
  return (
    <select value={color} onChange={e => setColor(e.target.value)}>
      <option value="red">Red</option>
      <option value="green">Green</option>
      <option value="blue">Blue</option>
    </select>
  );
}`;

export const bindingsSelectSvelte = `<script>
  let color = \$state('red');
<\/script>

<select bind:value={color}>
  <option value="red">Red</option>
  <option value="green">Green</option>
  <option value="blue">Blue</option>
</select>
<p>Chosen: {color}</p>`;

<ReactSvelte react={bindingsSelectReact} svelte={bindingsSelectSvelte} />

## ลองเล่น: form ขนาดเล็ก

export const bindingsCode = `<script>
  let name = \$state('');
  let email = \$state('');
  let subscribed = \$state(false);
  let plan = \$state('free');

  let summary = \$derived(
    name
      ? name + ' (' + email + ') — ' + plan + (subscribed ? ' + newsletter' : '')
      : 'Fill out the form above'
  );
<\/script>

<form onsubmit={e => e.preventDefault()}>
  <label>
    Name
    <input bind:value={name} placeholder="Your name" />
  </label>
  <label>
    Email
    <input type="email" bind:value={email} placeholder="you@example.com" />
  </label>
  <label>
    Plan
    <select bind:value={plan}>
      <option value="free">Free</option>
      <option value="pro">Pro</option>
      <option value="enterprise">Enterprise</option>
    </select>
  </label>
  <label class="check">
    <input type="checkbox" bind:checked={subscribed} />
    Subscribe to newsletter
  </label>
</form>

<p class="summary">{summary}</p>

<Diff title="React controlled inputs vs Svelte bind:">

- controlled input ของ React ต้องมี `value={state}` + `onChange={e => setState(e.target.value)}` — สองชิ้น
- Svelte `bind:value={state}` ทำทั้งสองอย่างใน directive เดียว — two-way binding โดยไม่ต้องมี handler
- `bind:checked` สำหรับ checkbox, `bind:value` สำหรับ text/number/email/select/textarea
- ปรัชญาของ React: **one-way data flow** (ชัดเจน) ส่วน Svelte: **two-way binding เป็นแบบ opt-in** (สะดวก)
- `bind:` เป็น directive ของ Svelte มันใช้ได้แค่กับ DOM element หรือ component ที่เปิดเผย prop แบบ `$bindable()` — ใช้กับค่า JS ทั่วไปไม่ได้
- React ไม่มีอะไรเทียบได้ตรง ๆ ที่ใกล้เคียงที่สุดคือ custom hook (`useInput`) ที่ return `{ value, onChange }`

</Diff>
