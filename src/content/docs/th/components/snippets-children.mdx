React ส่งเนื้อหา UI ให้ component ผ่าน `children` (JSX ที่อยู่ระหว่าง tag) หรือ render props (function ที่ return JSX) ส่วน Svelte 5 แทนที่ระบบ `<slot>` แบบเก่าด้วย **snippets** — ชิ้นส่วน template ที่นำกลับมาใช้ซ้ำได้ มีชื่อ และรับ parameter ได้ นิยามด้วย `{#snippet}` และ render ด้วย `{@render}` snippet ตัว default ยังคงชื่อ `children` อยู่ ดังนั้นการใช้งานพื้นฐานจึงรู้สึกคุ้นเคยทันที

## Default children

export const childrenReact = `// Card.jsx
function Card({ children }) {
  return <div className="card">{children}</div>;
}

// Usage:
<Card>
  <p>Hello from inside!</p>
</Card>`;

export const childrenSvelte = `<!-- Card.svelte -->
<script>
  let { children } = \$props();
<\/script>

<div class="card">
  {@render children()}
</div>

<!-- Usage in parent: -->
<!-- <Card><p>Hello from inside!</p></Card> -->`;

<ReactSvelte react={childrenReact} svelte={childrenSvelte} />

prop `children` คือ **snippet function** — คุณต้องเรียกมันด้วย `{@render children()}` ส่วน `{children}` ของ React เป็นค่า (value) แต่ `children` ของ Svelte เป็น snippet ที่เรียกได้ (callable)

## Named snippets (เหมือน named slot / render props)

export const namedReact = `// Dialog.jsx
function Dialog({ header, body, footer }) {
  return (
    <div className="dialog">
      <div className="dialog-header">{header}</div>
      <div className="dialog-body">{body}</div>
      <div className="dialog-footer">{footer}</div>
    </div>
  );
}

// Usage:
<Dialog
  header={<h2>Confirm</h2>}
  body={<p>Are you sure?</p>}
  footer={<button>OK</button>}
/>`;

export const namedSvelte = `<!-- Dialog.svelte -->
<script>
  let { header, body, footer } = \$props();
<\/script>

<div class="dialog">
  <div class="dialog-header">{@render header()}</div>
  <div class="dialog-body">{@render body()}</div>
  <div class="dialog-footer">{@render footer()}</div>
</div>

<!-- Usage in parent: -->
<!--
<Dialog>
  {#snippet header()}<h2>Confirm</h2>{/snippet}
  {#snippet body()}<p>Are you sure?</p>{/snippet}
  {#snippet footer()}<button>OK</button>{/snippet}
</Dialog>
-->`;

<ReactSvelte react={namedReact} svelte={namedSvelte} />

## Snippets ที่มี parameter (แพตเทิร์น render props)

snippet รับ argument ได้ — สิ่งนี้แทนที่แพตเทิร์น render-prop ของ React ได้อย่างสะอาดสะอ้าน:

export const renderPropsReact = `// List.jsx
function List({ items, renderItem }) {
  return (
    <ul>
      {items.map((item, i) => (
        <li key={i}>{renderItem(item)}</li>
      ))}
    </ul>
  );
}

// Usage:
<List
  items={fruits}
  renderItem={fruit => <strong>{fruit}</strong>}
/>`;

export const renderPropsSvelte = `<!-- List.svelte -->
<script>
  let { items, renderItem } = \$props();
<\/script>

<ul>
  {#each items as item}
    <li>{@render renderItem(item)}</li>
  {/each}
</ul>

<!-- Usage in parent:
<List {items}>
  {#snippet renderItem(fruit)}
    <strong>{fruit}</strong>
  {/snippet}
</List>
-->`;

<ReactSvelte react={renderPropsReact} svelte={renderPropsSvelte} />

## ลองเล่น: card ที่มี children

export const snippetsChildrenCode = `<script>
  let items = \$state(['Learn Svelte', 'Build something', 'Deploy to Vercel']);
<\/script>

{#snippet card(title, body)}
  <div class="card">
    <h3>{title}</h3>
    <p>{body}</p>
  </div>
{/snippet}

<h2>My Todo List</h2>
{#each items as item, i}
  {@render card('Task ' + (i + 1), item)}
{/each}

<style>
  .card { border: 1px solid #ccc; border-radius: 6px; padding: 0.75rem 1rem; margin-bottom: 0.5rem; }
  h3 { margin: 0 0 0.25rem; color: #FF3E00; }
  p { margin: 0; }
</style>`;

<Diff title="React children/render props vs Svelte snippets">

- React `children` (ค่าแบบ ReactNode) → Svelte snippet `children` (function ที่เรียกด้วย `{@render children()}`)
- named render props ของ React (function ที่ส่งเป็น props) → named snippet ของ Svelte (`{#snippet name()}` … `{/snippet}` ภายใน tag ของ component)
- แพตเทิร์น render prop ของ React `renderItem={item => <JSX />}` → snippet แบบรับ parameter ของ Svelte `{#snippet renderItem(item)}` … `{/snippet}`
- Svelte 4 `<slot name="header">` → Svelte 5 `{@render header()}` พร้อม named snippet prop
- `{@render snippet?.()}` — ใช้ optional chaining เมื่อ snippet prop อาจเป็น undefined

</Diff>
