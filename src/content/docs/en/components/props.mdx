In React you receive props as a function argument and destructure them in the parameter list. In Svelte 5 you call the `$props()` rune and destructure the result — the shape is nearly identical, so the mental mapping is immediate.

## React props vs `$props()`

export const propsReact = `// Badge.jsx
function Badge({ label, count = 0, color = 'blue' }) {
  return (
    <span style={{ color }}>
      {label}: {count}
    </span>
  );
}
export default Badge;`;

export const propsSvelte = `<!-- Badge.svelte -->
<script>
  let { label, count = 0, color = 'blue' } = \$props();
<\/script>

<span style="color: {color}">
  {label}: {count}
</span>`;

<ReactSvelte react={propsReact} svelte={propsSvelte} />

The destructuring syntax is **identical**. Default values work the same way. The only difference is the source: React uses the function argument; Svelte uses `$props()`.

## Rest props

If you want to forward unknown attributes to a DOM element (e.g., for a wrapper component), spread the rest:

export const propsRestReact = `function Button({ children, ...rest }) {
  return <button {...rest}>{children}</button>;
}`;

export const propsRestSvelte = `<script>
  let { children, ...rest } = \$props();
<\/script>

<button {...rest}>
  {@render children?.()}
</button>`;

<ReactSvelte react={propsRestReact} svelte={propsRestSvelte} />

<Diff title="React props vs Svelte $props()">

- React: `function MyComp({ name, count = 0 }) { ... }` — props come from the function parameter.
- Svelte: `let { name, count = 0 } = $props()` — same destructuring, from the rune instead.
- Both support default values with `= value` syntax.
- Both support rest spread with `...rest` for forwarding unknown attributes.
- TypeScript: React uses `FC<Props>` or typed parameters; Svelte uses `interface Props` + `$props<Props>()`.
