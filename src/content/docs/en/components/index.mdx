In React every component is a **function** that returns JSX. In Svelte every component is a **`.svelte` file** that combines script, markup, and styles in a single, familiar HTML-like format. No JSX, no `className`, no separate CSS file needed.

This module walks you through the core building blocks:

| Lesson | Concept |
|---|---|
| This page | `.svelte` file structure vs React function components |
| Props | `$props()` rune vs React props destructuring |
| Template blocks | `{#if}`, `{#each}`, `{#await}` vs JSX ternaries and `.map()` |
| Events | `onclick={fn}` vs `onClick={fn}` |
| Snippets & children | `{#snippet}` / `{@render}` vs React `children` |
| Bindings | `bind:value` vs controlled inputs |
| Styles | Scoped `<style>` vs CSS Modules / CSS-in-JS |

## A `.svelte` file has three sections

```
<script>   ← component logic (JS/TS, runes)
</script>

markup     ← HTML template (no JSX, no "return")

<style>    ← scoped CSS (auto-scoped to this component)
</style>
```

Only the markup section is required. `<script>` and `<style>` are optional.

## React component vs Svelte component

export const indexReact = `// Greeting.jsx
function Greeting({ name }) {
  return <h1>Hello, {name}!</h1>;
}
export default Greeting;`;

export const indexSvelte = `<!-- Greeting.svelte -->
<script>
  let { name } = \$props();
<\/script>

<h1>Hello, {name}!</h1>`;

<ReactSvelte react={indexReact} svelte={indexSvelte} />

The markup in a `.svelte` file is the entire file body — no `return` statement, no parentheses, no JSX pragma.

## Try it: your first Svelte 5 component

export const indexCode = `<script>
  let name = \$state('React developer');
<\/script>

<h2>Hello, {name}!</h2>
<input bind:value={name} placeholder="Type your name" />

<style>
  h2 { color: #FF3E00; }
  input { margin-top: 0.5rem; padding: 0.25rem 0.5rem; }
</style>`;

<Diff title="React component vs Svelte component">

- React: a **function** that `return`s JSX. Must import React (or use JSX transform).
- Svelte: a **`.svelte` file**. Markup is the template — no `return`, no JSX.
- React: CSS via CSS Modules, Styled Components, or `className`.
- Svelte: `<style>` block is **automatically scoped** to this component.
- React: `export default function MyComponent`.
- Svelte: the file itself _is_ the export — no `export default` needed.

</Diff>
