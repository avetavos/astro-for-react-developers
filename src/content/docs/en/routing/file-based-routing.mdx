In react-router you declare routes in JavaScript. In Next.js you create files in a `pages/` or `app/` directory. SvelteKit works like Next.js App Router: **create a folder, add `+page.svelte`, and you have a route.** No imports, no route config, no wrapping component needed.

> Run this in a SvelteKit project (`npx sv create`).

## Basic routes

The simplest case: a static path maps directly to a folder + `+page.svelte`.

export const basicReact = `// react-router v6
import { createBrowserRouter } from 'react-router-dom';
import About from './pages/About';

const router = createBrowserRouter([
  { path: '/about', element: <About /> },
]);`;

export const basicSvelte = `<!-- src/routes/about/+page.svelte -->
<!-- This file IS the /about route. No config needed. -->

<h1>About</h1>
<p>Welcome to the about page.</p>`;

<ReactSvelte react={basicReact} svelte={basicSvelte} reactTitle='React (react-router)' svelteTitle='SvelteKit' />

## Dynamic routes: `[param]`

react-router uses `:id` in the path string. Next.js uses `[id]` in the filename. SvelteKit uses `[param]` as a **folder name** ‚Äî the folder name (without brackets) becomes the parameter key.

export const dynReact = `// react-router v6
{ path: '/posts/:id', element: <PostPage /> }

// Next.js Pages Router
// pages/posts/[id].tsx

// Next.js App Router
// app/posts/[id]/page.tsx`;

export const dynSvelte = `<!-- src/routes/posts/[id]/+page.svelte -->
<!-- Matches /posts/1, /posts/hello, /posts/anything -->

<h1>Post detail page</h1>`;

<ReactSvelte react={dynReact} svelte={dynSvelte} reactTitle='React / Next.js' svelteTitle='SvelteKit' />

## Route file-tree walkthrough

A realistic SvelteKit `src/routes/` structure:

```
src/routes/
‚îú‚îÄ‚îÄ +page.svelte          ‚Üê /
‚îú‚îÄ‚îÄ about/
‚îÇ   ‚îî‚îÄ‚îÄ +page.svelte      ‚Üê /about
‚îú‚îÄ‚îÄ blog/
‚îÇ   ‚îú‚îÄ‚îÄ +page.svelte      ‚Üê /blog
‚îÇ   ‚îî‚îÄ‚îÄ [slug]/
‚îÇ       ‚îî‚îÄ‚îÄ +page.svelte  ‚Üê /blog/:slug
‚îî‚îÄ‚îÄ dashboard/
    ‚îú‚îÄ‚îÄ +layout.svelte    ‚Üê shared layout
    ‚îú‚îÄ‚îÄ +page.svelte      ‚Üê /dashboard
    ‚îî‚îÄ‚îÄ settings/
        ‚îî‚îÄ‚îÄ +page.svelte  ‚Üê /dashboard/settings
```

Every `+page.svelte` is a route. The `+layout.svelte` inside `dashboard/` wraps all pages in that folder and its subfolders ‚Äî equivalent to a `<Outlet>` layout route in react-router or a `layout.tsx` in Next.js App Router.

## Accessing route params

### Via the `load` function (recommended)

The cleanest approach is to read params inside `+page.js` or `+page.server.js` and pass them to the component as `data`. The component stays param-agnostic.

```js
// src/routes/blog/[slug]/+page.js
export async function load({ params }) {
  const post = await fetchPost(params.slug);
  return { post };
}
```

```svelte
<!-- src/routes/blog/[slug]/+page.svelte -->
<script>
  let { data } = \$props();
  // data.post is what load() returned
<\/script>

<h1>{data.post.title}</h1>
<article>{@html data.post.content}</article>
```

### Via the `$page` store (direct access)

You can also read params directly in the component using the `$page` store from `$app/state`:

```svelte
<!-- src/routes/blog/[slug]/+page.svelte -->
<script>
  import { page } from '\$app/state';
  // page.params.slug is the current [slug] value
<\/script>

<p>Viewing slug: {page.params.slug}</p>
```

The `load` function approach is preferred ‚Äî it keeps data fetching co-located and typed, and it works well with server-side rendering.

<Diff title="react-router / Next.js params vs SvelteKit params">

- react-router: dynamic segment syntax is `:id` in the path string. Access via `useParams()` hook.
- Next.js Pages Router: filename `[id].tsx`. Access via `useRouter().query.id`.
- Next.js App Router: folder `[id]/page.tsx`. Access via `props.params.id` (async in Next 15+).
- SvelteKit: folder `[id]/+page.svelte`. Access via `params.id` in `load()`, or `page.params.id` from `$app/state`.
- SvelteKit param folders use `[square-brackets]` just like Next.js, not `:colons` like react-router.
- SvelteKit layouts (`+layout.svelte`) auto-wrap child routes ‚Äî no `<Outlet />` needed inside the layout.

</Diff>
